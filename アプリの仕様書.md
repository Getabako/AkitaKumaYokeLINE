# neoクマダス（仮）アプリ仕様書

> **コンセプト（v0.2で更新）：bearbell/クマダスと"同じ土俵で殴り合わない"。**
> 目撃情報の速さ競争（過去の点を速く出す）はクワガタ（bearbell）の土俵。そこは避ける。
> こちらは **LINEミニアプリ**で、①DL不要で家族みんなに即届く到達力、②学校・子ども・交通まで含む「地域お守り」、
> ③**クマに"遭わない暮らし方"＝予防の知識**、という別の軸で勝つ。クマは"入口"、本体は家族と暮らしの安心。
> ※方針転換の経緯と実装状況は末尾「12. 実装状況と方針アップデート」を参照。

- 文書バージョン: v0.2（2026-06-22 構築反映）
- 作成日: 2026-06-22
- 想定提案先: 秋田県内自治体／県議会議員ルート
- 想定予算規模: 200〜300万円（初期開発フェーズ）

## 1. なぜ作るのか（背景）

秋田はクマ被害が全国最悪レベルで、行政・住民の関心が極めて高い。すでに2つの主要プレイヤーがいる。

| | クマダス | bearbell / クマップ |
|---|---|---|
| 運営 | 秋田県（公式） | AIU発スタートアップ（株式会社BearBell） |
| 開始 | 2024年7月〜運用中 | 2026年6月、秋田市で実証実験中（〜6/30） |
| データ | 市町村・警察の情報を一元化 | 全国約1,700自治体のオープンデータ自動収集＋市民投稿＋カメラ画像認識AI |
| 通知 | エリア指定の**メール**配信 | 投稿から**5秒**で共有・AI信頼度スコア付き |
| 形式 | Webサイト（要会員登録） | 独立スマホアプリ（要DL） |
| 弱点 | UIが古い／通知がメール／速報性に難 | アプリDLの障壁／普及はこれから／家族間共有が弱い |

### 両者の弱点は「LINEミニアプリ化」で全部埋まる

1. **DLの壁がない** — 県民の大多数がすでにLINEを使っている。友だち追加だけで使える。
2. **通知がメールでなくLINEプッシュ** — 開封率・即時性がメールと段違い。命に関わる速報こそLINE。
3. **家族・学校・地域でそのまま共有** — トーク転送・グループ連携がLINEの本質。「通学路にクマ」を親が30秒で共有できる。
4. **議員が欲しい情報を全部のせられる** — クマだけでなく、学校連絡・保護者向け・交通系まで地域安全のハブにできる。

> **狙い：クマダス（公式の信頼性）× bearbell（技術の速さ）の上位互換を、最も普及している土管＝LINEで実装する。**

- **第一次：** 秋田県内の住民（特に山際・郊外の生活者、高齢者、農家）

## 3. 機能要件

### 3-1. コア機能（クマ・鳥獣出没）— クマダス＋bearbellの統合

- **リアルタイム出没マップ**
  - 地図上に目撃・痕跡・人身被害をピン表示（クマダス相当）
  - 投稿から数秒で反映（bearbell相当の速報性）
  - 獣種フィルタ：ツキノワグマ／イノシシ／ニホンジカ／サル
  - 種別フィルタ：目撃／人身被害／食害・痕跡／その他
  - 期間・市町村・現在地からの半径（10/20/30km）で絞り込み
- **エリア指定プッシュ通知（LINEの主役機能）**
  - 自宅・職場・実家・通学路など複数地点を登録
  - 登録エリアで出没が入ると即LINE通知
  - 重大度別（注意報／警報／人身被害は最優先＝強通知）
- **住民投稿**
  - 位置（GPS自動）＋写真＋ひとことで30秒投稿
  - **AI信頼度スコア**：投稿内容・写真をAIが判定し信頼度を表示（bearbell相当、偽情報対策）
  - 重複投稿の自動マージ
- **行政・公式データ連携**
  - クマダス／自治体オープンデータの自動取り込み（可能な範囲でAPI・スクレイピング）
  - 自治体・警察が直接入力できる管理画面

### 3-2. 学校・保護者機能（議員の要望ライン）

- **通学路アラート**：学校・通学路を登録した保護者に、その経路周辺の出没を優先通知
- **学校からの一斉連絡**：休校・集団下校・見守り要請を教育委員会／学校から配信
- **見守り共有**：家族グループで子どもの登下校時間帯に絞った警戒モード

### 3-3. 交通系情報（議員の要望ライン）

- **道路・公共交通の地域情報**：通行止め・冬季閉鎖・バス運休・除雪情報などを同一マップ／通知で配信
- クマ出没による道路規制との連動表示（例：「この道、クマ＋通行止め」）
- ※自治体・交通事業者のデータ連携が前提。第2フェーズで拡張。

### 3-4. 安全補助機能（bearbell由来）

- **クマよけ鈴モード（任意）**：歩行中にバックグラウンドで鈴音・音声アナウンス（bearbellの売り）
  - ※LINEミニアプリのバックグラウンド音声制約があるため、実現可否は技術検証マター。MVPでは「外部リンク or 簡易版」で逃がす選択肢あり。
- **緊急時の行動ガイド**：クマに遭遇した時の対処をワンタップ表示

### 3-5. 管理・運用機能

- 自治体・学校向け管理ダッシュボード（投稿の承認・公式情報入力・配信）
- 投稿モニタリング（AI信頼度＋人手の二段構え）
- 統計・ヒートマップ（出没傾向の可視化＝行政の政策資料になる）

## 5. 技術スタック（ifJukuManager標準に準拠）

- **フロント：** LINEミニアプリ（LIFF）／React 19 + TypeScript + Vite + Tailwind CSS 3.4.17 + Zustand
- **地図：** Google Maps API もしくは MapLibre（コスト次第で選定）
- **バックエンド：** Vercel Functions（Serverless）
- **DB：** Neon PostgreSQL + Prisma 5.22.0（位置情報はPostGIS拡張を検討）
- **通知：** LINE Messaging API（プッシュ）
- **AI判定：** Gemini API（投稿テキスト・画像の信頼度スコアリング）
- **認証：** LINE LIFF
- **デプロイ：** Vercel（GitHub: Getabako organization）
- **データ取込：** 定期バッチ（クマダス／自治体オープンデータ）

> ※既存LINEMiniApp群の構成をそのまま流用できるので、開発スピードとコストで圧倒的に有利。

## 6. 画面構成（ラフ）

1. **ホーム／マップ** — 現在地中心の出没マップ。重大アラートを上部にバナー表示
2. **投稿** — カメラ起動→位置自動→ひとこと→送信（最短2タップ）
3. **通知設定** — エリア登録・重大度別ON/OFF・通学路登録
4. **一覧／検索** — フィルタ付きリスト
5. **詳細** — ピン詳細（写真・日時・信頼度・周辺情報）
6. **お知らせ** — 自治体・学校からの公式配信
7. **設定** — プロフィール・登録地点・遭遇時ガイド
8. **（管理者用）ダッシュボード** — 入力・承認・統計

## 7. データ連携・運用上の論点（要確認事項）

- クマダスの公式データを正式に連携できるか（県との交渉）。難しければ公開情報の取り込み＋独自投稿で先行。
- bearbell/クマップが既に秋田市で実証中。**競合か、棲み分けか、巻き取りか**の戦略整理が必要。
- 学校・交通データは自治体・事業者の協力が前提 → 議員ルートの価値がここで効く。
- 個人情報・投稿の取り扱い（位置情報・写真）の規約整備。
- 偽情報対策＝AI信頼度＋通報＋（必要なら）緩い本人確認。**IP表示等の晒し系はやらない**（プライバシー・行政炎上リスク）。

## 12. 実装状況と方針アップデート（2026-06-22 構築）

### 方針転換（なぜ「ずらす」のか）
- bearbell（クワガタ髪の学生起業家）は**ピッチ・調達・政治が非常に強い**（起業家甲子園、総務省ICTスタートアップリーグ採択、東証グロース上場のGxPと技術連携、AIUバック、秋田市で行政参加の実証実験6/1〜30）。
- 正面（同じ自治体予算・目撃速度競争）で殴り合うのは不利。製品で勝てても政治で負ける。
- → **土俵をずらす**。①LINEでの到達力（DL不要・家族即共有）②子ども・暮らしの地域お守り③"遭わない暮らし方"の予防知識。
- bearbellの目撃通知機能は「クマ回避に効果が薄い」という判断のもと、**模倣しない**。クマ機能は最小限（地図＋通報＋撃退）に留め、本体価値を家族・暮らし・予防に置く。

### アーキテクチャ（実装したもの）
- **LINE公式アカウント1つ＋リッチメニュー（6マス）**を入口に、機能を**5つのLIFFミニアプリ**に分割。
- **line-harness-oss-main（MIT・無料運用のLステップ代替）**で「登録エリアでクマ出没→公式アカウントから自動push」を擬似ジオ配信（エリアタグ＋受信Webhook＋オートメーション）で実現。
- 各アプリは ai-diagnosis と同一作法（React19 + TS + Vite + Tailwind3.4.17 + Zustand + LIFF）。高齢者向けに基準17px・行間1.9・特大ボタンを徹底。
- 地図は素のLeaflet＋OpenStreetMap（APIキー不要）。出没データは `VITE_API_BASE` があれば共有API、無ければデモ（モック＋localStorage）で即動作。

### 構築した5アプリ（neoKumadasu/ 配下）
| フォルダ | 機能 | 状態 |
|---|---|---|
| kuma-map | クマ出没マップ（地図で見る・フィルタ） | 雛形完成・ビルド確認済 |
| kuma-report | 目撃をしらせる（GPS自動取得・本人確認文言・最短通報） | 雛形完成 |
| kuma-bakuon | クマ撃退ブザー（Web Audio合成・熊鈴モード） | 雛形完成 |
| kuma-alert | 通知エリア設定（市町村登録→通知） | 雛形完成 |
| kotsu-info | 交通・くらし情報（通行止め/除雪/バス/クマ規制） | 雛形完成 |

### 追加で固まった"本体価値"
- **クマに遭わない暮らし方ガイド**（`クマに遭わない暮らし方ガイド.md`）。遭わない6条件（時間・季節・場所・天気・行動・誘引源）。
- 将来機能 **「今日のクマ遭遇リスク」**：時間帯×季節×天気×堅果豊凶×周辺目撃 で動的にリスクを平易表示。これが目撃ピン競争と別軸の差別化の核。
- リッチメニュー6マス目「つかいかた」を、この遭わない暮らし方ガイドにする想定。
