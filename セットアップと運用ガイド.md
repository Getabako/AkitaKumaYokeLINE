# あきた見守りLINE — セットアップと運用ガイド

このフォルダ一式を「動くLINE公式アカウント」にするための手順書。
**LINEの設定をやったことがない前提**で、最初から順番に書いてある。上から順にやればいい。

---

## 0. 全体像（これだけ先に頭に入れる）

```
                ┌──────────────────────────────┐
                │   LINE公式アカウント（1つ）   │
                │   ＝ 友だち追加してもらう窓口  │
                └───────────────┬──────────────┘
                                │
                  下に「リッチメニュー」を出す
                                │
   ┌──────────┬──────────┬──────────┬──────────┬──────────┬────────┐
   │ クマ地図  │目撃しらせ │撃退ブザー │通知エリア │交通くらし │つかい方 │
   └────┬─────┴────┬─────┴────┬─────┴────┬─────┴────┬─────┴────────┘
        │          │          │          │          │
     LIFFで開く  LIFFで開く  LIFFで開く  LIFFで開く  LIFFで開く
        │          │          │          │          │
   ┌────▼───┐ ┌───▼────┐ ┌───▼─────┐ ┌──▼─────┐ ┌──▼──────┐
   │kuma-map│ │kuma-   │ │kuma-    │ │kuma-   │ │kotsu-   │
   │        │ │report  │ │bakuon   │ │alert   │ │info     │
   └────────┘ └────────┘ └─────────┘ └────────┘ └─────────┘
   （5つのミニアプリ。それぞれ Vercel に置く）

   ＋ line-harness（Lステップ的な裏方）
     ＝ 「登録エリアでクマが出たら公式アカウントから自動通知」を担当
```

ポイントは3つだけ：
1. **公式アカウントは1つ**。その下の**リッチメニュー（6マスのボタン画像）**が入口。
2. リッチメニューの各マスを押すと、**5つのミニアプリ（LIFF）**のどれかが開く。
3. **line-harness** が「エリア登録した人に、クマ通知を自動で送る」裏方をやる。

各アプリは中身が同じ作法（React + TypeScript + Vite + Tailwind + LIFF）。
**まず手元で動かしてみる**のが一番わかりやすい：

```bash

npm install
npm run dev
# → ブラウザで http://localhost:5173 が開く。LIFF未設定でも「デモ動作」する作りにしてある
```

5つとも同じ手順で動く（`kuma-report` `kuma-bakuon` `kuma-alert` `kotsu-info` も同様）。
※デモ動作＝地図やデータがモック（仮データ）で表示される。本番データは後で繋ぐ。

## 3. 手順A：LINE公式アカウントを作る

1. [LINE Official Account Manager](https://manager.line.biz/) にログイン → 「アカウントを作成」。
2. アカウント名（例：あきた見守り）・業種を入れて作成。
3. これで「友だち追加」してもらう窓口ができた。QRコードはあとで配布に使う。

---

## 4. 手順B：Messaging API と LINE Login を有効にする

LINE Developers（[developers.line.biz](https://developers.line.biz/)）で設定する。
※ここが一番つまずきやすいので丁寧に。

### B-1. プロバイダーを作る
- コンソール → 「Create a new provider」→ 名前（例：あきた見守り）。

### B-2. Messaging APIチャネルを作る（公式アカウントと連携）
- 「Create a Messaging API channel」を選ぶ。すでに作った公式アカウントと紐づける案内に従う。
- できたら以下を控える（あとで使う）：
  - **チャネルシークレット**（Basic settings）
  - **チャネルアクセストークン（長期）**（Messaging API タブで発行）

### B-3. LINE Loginチャネルを作る（LIFF＝ミニアプリに必須）
- 同じプロバイダーで「Create a LINE Login channel」。
- これが無いとミニアプリ（地図や目撃登録）が開けない。

---

## 5. 手順C：5つのミニアプリをVercelに公開して、LIFFを登録する

各アプリ（kuma-map など）を1個ずつ次の流れで公開する。**5回繰り返す**。

### C-1. Vercelに公開してURLをもらう
- 一番簡単：各アプリフォルダで `npx vercel`（初回はログインと数問の質問）。
- もしくは GitHub（Getabako org）にpushして Vercel と連携（自動デプロイ）。
- 公開されると `https://kuma-map-xxxx.vercel.app` のようなURLがもらえる。これを控える。

### C-2. LIFFアプリを登録してLIFF IDをもらう
- LINE Developers → さっきの **LINE Loginチャネル** → 「LIFF」タブ → 「Add」。
  - エンドポイントURL：C-1のVercel URL（例 `https://kuma-map-xxxx.vercel.app`）
  - サイズ：**Full**（全画面。地図やボタンを大きく使うため）
  - Scope：`profile`, `openid` にチェック
- 登録すると **LIFF ID**（`1234567890-abcdEfgh` の形）がもらえる。

### C-3. アプリにLIFF IDを設定して再デプロイ
- 各アプリの `.env`（`.env.example` をコピーして作る）に：
  ```
  VITE_LIFF_ID=ここにLIFF ID
  VITE_API_BASE=     # 後述の共有バックエンドのURL。最初は空でOK（デモ動作）
  ```
- Vercel側にも同じ環境変数を登録（Vercel Dashboard → Settings → Environment Variables）。
- 再デプロイすれば、そのアプリは「LINEの中で本人確認つきで動く」状態になる。

**これを5アプリぶん繰り返す。** 5つのLIFF IDとVercel URLが手元に揃えばOK。

> メモ：各アプリは「LIFF URL」＝ `https://liff.line.me/{LIFF ID}` でも開ける。
> リッチメニューのボタンには、この LIFF URL を割り当てる（手順D）。

## 6. 手順D：リッチメニュー（6マスのボタン）を作る

公式アカウントの下に出る、あの画像のメニュー。

1. [LINE Official Account Manager](https://manager.line.biz/) → 対象アカウント → 「トークルーム管理」→「リッチメニュー」→「作成」。
2. **表示期間**：今日〜ずっと先、にしておく。
3. **コンテンツ設定**：
   - テンプレート：**大（6分割：2行×3列）** を選ぶ。
   - 背景画像：`assets/richmenu.png` をアップロード（2500×1686で用意済み）。
4. **各マスのアクション**を設定（画像の並びと合わせる）：

   | マス | ラベル | アクション | リンク先 |
   |---|---|---|---|
   | 左上 | クマ地図 | リンク | `https://liff.line.me/{kuma-map の LIFF ID}` |
   | 中上 | 目撃をしらせる | リンク | `https://liff.line.me/{kuma-report の LIFF ID}` |
   | 右上 | クマ撃退ブザー | リンク | `https://liff.line.me/{kuma-bakuon の LIFF ID}` |
   | 左下 | 通知エリア設定 | リンク | `https://liff.line.me/{kuma-alert の LIFF ID}` |
   | 中下 | 交通・くらし情報 | リンク | `https://liff.line.me/{kotsu-info の LIFF ID}` |
   | 右下 | つかいかた | リンク or テキスト | 使い方ページ or 「メニューを押すと各機能が開きます」等 |

5. 保存して「表示する」。これで友だちのトーク画面の下に6マスメニューが出る。

## 7. 手順E：line-harness で「クマ通知」を動かす（Lステップ的な裏方）

「自分の登録エリアでクマが出たら、公式アカウントから自動でLINE通知」を実現する部分。
（場所：`/Users/takasaki19841121/Desktop/line-harness-oss-main`。MITライセンス・無料運用）

### 仕組み（先に理解）
LINEには「GPSで半径◯km内の人に自動配信」という機能は**無い**。だから次の方式で擬似的に実現する：

```
① ユーザーが kuma-alert で「秋田市」を登録
     → line-harness 側で、そのユーザーに「クマ注意-秋田市」タグを付ける
② どこかでクマ出没（kuma-report の投稿 or 行政データ）
     → line-harness の「受信Webhook」にクマ出没を通知（市町村つき）
③ オートメーションが発火
     → 「クマ注意-秋田市」タグを持つ人だけに、公式アカウントからpush通知
```

### セットアップ（ざっくり）
```bash
cd /Users/takasaki19841121/Desktop/line-harness-oss-main
npx create-line-harness     # 対話式。Cloudflareログイン→DB作成→デプロイまで自動（約5分）
```
途中で、手順Bで控えた値を聞かれる：
- `LINE_CHANNEL_SECRET` / `LINE_CHANNEL_ACCESS_TOKEN`（Messaging API）
- `LINE_LOGIN_CHANNEL_ID` / `LINE_LOGIN_CHANNEL_SECRET`（LINE Login）
- `API_KEY`（自分で決める管理用の合言葉）

デプロイ後、LINE Developers の Messaging API → **Webhook URL** に
`https://{あなたのworker}.workers.dev/webhook` を設定（友だち登録などを自動で拾うため）。

### クマ通知の配線（line-harness 側でやること）
1. **エリアのタグを作る**（例：「クマ注意-秋田市」を25市町村ぶん、または使う地域だけ）。
2. **kuma-alert の保存先を line-harness にする**：
   - 今は各アプリの `VITE_API_BASE` を空にしてデモ動作。
   - 本番は、`VITE_API_BASE` を共有バックエンド（後述）にし、kuma-alert の登録 → 該当ユーザーにエリアタグを付与、と繋ぐ。
3. **クマ出没の入口（受信Webhook）を作る**：line-harness で「受信Webhook」を1本作る。
   - kuma-report の投稿時、または行政データ取り込みバッチから、この Webhook に
     `{ "location": "秋田市", "severity": "high" }` のようにPOSTする。
4. **オートメーションを作る**：受信Webその受信をトリガーに、
   「`クマ注意-{location}` タグを持つ友だちへ send_message（push）」を実行するルールを設定。

これで「②でクマ出没→③で該当エリアの人に自動通知」が回る。
（詳細APIは `line-harness-oss-main/docs/` 参照。curl例も仕様書に載せてある。）

## 8. 共有バックエンド（地図と目撃登録のデータをつなぐ）

`kuma-map`（見る）と `kuma-report`（送る）は、本来**同じデータ**を見る必要がある。
今は各アプリが「デモ動作（端末内のローカル保存＋仮データ）」なので、別々に見えている。

本番では、**1つの共有API**を立てて、両方の `VITE_API_BASE` にそのURLを入れる：

- 例：`https://neokuma-api.vercel.app`
- 必要なエンドポイント：
  - `GET  /api/sightings` … 出没一覧を返す（kuma-map が読む）
  - `POST /api/sightings` … 出没を1件登録（kuma-report が書く）
  - `POST /api/alert-prefs` … 通知エリア設定を保存（kuma-alert が書く）
  - `GET  /api/info` … 交通くらし情報を返す（kotsu-info が読む）
- データベースは Neon PostgreSQL + Prisma（このリポジトリの標準）を推奨。
  - ※この共有APIは未実装。MVPでは「まず1〜2アプリをデモで議員に見せる→予算が付いたら共有APIを実装」でよい。

## 11. つまずきやすいポイント

- **ミニアプリが真っ白**：`VITE_LIFF_ID` 未設定 or LIFFエンドポイントURLとVercel URLが違う。→ C-2/C-3を確認。
- **LINEの外（PCブラウザ）で開くとログインループ**：LIFFは基本LINEアプリ内で開く前提。テストはスマホのLINEから。
- **通知が来ない**：line-harness のWebhook URL未設定 or エリアタグが付いていない。→ 手順E。
- **音が鳴らない（爆音アプリ）**：iOSは最初のタップで音が有効化される。1回タップしてから。端末の音量も上げる。
- **地図のピンが出ない**：`VITE_API_BASE` が空ならデモデータが出る。出ない時はブラウザのコンソールを確認。
